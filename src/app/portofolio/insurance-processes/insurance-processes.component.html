Automated Insurance Processes

Characteristics

Project Type: Enterprise

Client: Cegeka, Germany

Period: July 2022 - May 2023

Overview

       The project involved working for a client based in Germany with the goal of optimizing and automating their insurance processes. The objective was to develop a streamlined system that automates workflows, generates documents, enables document signing, stores documents, and sends regular notifications to customers.

       The project involved the development of a system composed of more than 7 microservices and one monolithic web service.

Key Features

  • Process Automation: Automated system to initiate and streamline insurance processes.
  • Microservices Architecture: Migrated from a monolithic system to microservices for scalability and modularity.
  • Monolith Maintenance: Ensured the functional, stable, and compatible operation of the existing monolithic system throughout the transition phase until complete decomposition into microservices was achieved.
  • Documents Storage and Generation: Generated and stored insurance documents using templates and customer information.
  • Notification System: Sent daily updates and reminders to customers regarding their insurance processes.

Responsibilities

  • Development cycle: Involved in all phases of the software development lifecycle following the Scrum methodology.
  • High-Quality Code: Wrote reusable and maintainable code adhering to best practices and coding standards.
  • Microservices architecture: Designed and implemented microservices using Domain-Driven Design and CQRS patterns.
  • Code Review and Quality Assurance: Participated in code reviews and ensured adherence to standards.
  • Issue Resolution: Troubleshot and resolved issues in microservices and monolith applications.
  • Enhanced front-end: Improved the UI using Angular framework, implementing enhancements and fixing bugs.
  • Documentation: Created technical documentation including architecture diagrams, API documentation, and user guides.

Achievements

  • Achieved a 30% reduction in manual intervention by developing an automated system for initiating insurance processes and streamlining workflows.
  • Successfully migrated the monolithic insurance system to a microservices architecture, resulting in a 40% improvement in scalability, modularity, and maintainability.
  • Efficiently resolved 90% of issues in the microservices and monolith applications, minimizing downtime and ensuring smooth software operation.
