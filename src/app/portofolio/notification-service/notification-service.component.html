Notification Service

Characteristics

Project Type: Enterprise

Client: OpenLM, Israel

Period: January 2022 - July 2022

Overview

       The Notification Service is a comprehensive and versatile solution designed to facilitate the sending of notifications. It consists of 4 interconnected microservices, each serving a specific purpose. These microservices are:

  • Notification Gateway: This microservice works as an orchestrator directs requests to the appropriate notification service, supporting multiple types of requests such as batch, bulk, single, on-demand, and multiple notifications.
  • Email Notifications: This microservice sends customized emails with preset or custom designs, including personalized email names.
  • Ticketing Notification: This microservice creates tickets within customer ticketing systems, automating the process and improving efficiency.
  • Push Notifications: This microservice generates push notifications for front-end application services, ensuring effective user engagement across mobile and web platforms.

Responsibilities

  • Design and Planning: I spearheaded the design phase, creating a comprehensive solution architecture that encompassed the system structure, data flows, and integration points.
  • Development and Implementation: I actively participated in the development process, taking responsibility for ensuring strict adherence to coding standards and best practices.
  • Deployment and Maintenance: Working closely with the IT team to ensure a seamless deployment process. Handling ongoing maintenance tasks, including bug fixes, performance optimization, and feature updates.

Achievements

  • Enhanced user engagement across web platforms by implementing the Push Notifications microservice, which resulted in a significant 20% increase in user interactions and retention.
  • Successfully personalized email communication by implementing the Email Notifications microservice, resulting in a 25% increase in customer engagement and satisfaction.
  • Achieved high code reusability as a generic mechanism that can be utilized across all services. This approach allowed for efficient development and reduced redundancy, resulting in time and cost savings for future projects.
